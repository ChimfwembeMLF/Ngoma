# Implementation Plan: Artist Tipping

**Branch**: `007-artist-tipping` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/007-artist-tipping/spec.md`

## Summary

Deliver the **second Phase 2 Week 15вЂ“16 slice** (`PROJECT REQUIREMENTS.md` В§11.2): **artist tipping**. Extend the existing `payments` module with `TIP` purpose, a new `tips` table, tip initiation/completion logic (95% artist share), artist tip list API, and client tip page reachable from track detail. Reuse PawaPay/sandbox deposit flow and existing analytics earnings aggregation.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (api + client workspaces)

**Primary Dependencies**:
- API: NestJS 11+, TypeORM (`Payment`, `Earnings`, `Artist`, new `Tip` entity)
- Client: React 18, Vite, TanStack Query, design system (003/004)
- Payments: Mirror `initiateDeposit` + `completePayment` patterns from `payments.service.ts`

**Storage**: PostgreSQL 15+ вЂ” migration adds `tips` table, extends `payments.purpose` and `earnings.source` CHECK constraints, nullable `earnings.track_id`

**Testing**: Manual VS-701вЂ“VS-704 per `quickstart.md`

**Target Platform**: Web SPA + REST API

**Project Type**: Yarn monorepo вЂ” `api/` + `client/`

**Performance Goals**: Tip initiation O(1); artist tip list paginated, default 20

**Constraints**:
- Extend `api/src/modules/payments/` вЂ” no separate payments provider
- Routes under `/api/v1/payments/tip` and `/api/v1/tips/received`
- Platform fee **5%** for tips (vs 30% for downloads)
- Smallest diff вЂ” one migration, tip entity in payments module

**Scale/Scope**: 1 migration, ~10 API files, ~5 client files, 1 new page route

**Reference**: `PROJECT REQUIREMENTS.md` В§3.6.1 tipping UI, В§2407 `tips` table, В§1991 TIP earnings split
