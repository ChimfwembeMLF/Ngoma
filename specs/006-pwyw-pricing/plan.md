# Implementation Plan: Pay What You Want Pricing

**Branch**: `006-pwyw-pricing` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/006-pwyw-pricing/spec.md`

## Summary

Deliver the **first Phase 2 Week 15–16 slice** (`PROJECT REQUIREMENTS.md` §11.2): **Pay What You Want (PWYW)** track pricing. Extend the existing `tracks` module and `payments` flow — no new NestJS module. Add `PAY_WHAT_YOU_WANT` to `PricingType`, a `min_price` column via TypeORM migration, validation in track DTOs and `payments.service.ts`, and client updates to upload form, track detail, discover cards, and checkout amount input.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (api + client workspaces)

**Primary Dependencies**:
- API: NestJS 11+, TypeORM (`Track`, `Payment`, `Earnings` — existing)
- Client: React 18, Vite, TanStack Query, design system (003/004)
- Payments: Existing `initiateDeposit` + PawaPay/sandbox completion path

**Storage**: PostgreSQL 15+ — migration adds `min_price` column and expands `pricing_type` CHECK constraint

**Testing**: Manual VS-601–VS-604 per `quickstart.md`

**Target Platform**: Web SPA + REST API

**Project Type**: Yarn monorepo — `api/` + `client/`

**Performance Goals**: No new endpoints; checkout validation is O(1) track lookup

**Constraints**:
- Extend `api/src/modules/tracks/` and `api/src/modules/payments/` — no new module
- Reuse `/api/v1/payments/deposit` with server-side min validation
- Smallest diff — one migration, enum extension, targeted UI changes

**Scale/Scope**: 1 migration, ~8 API files touched, ~6 client files touched

**Reference**: `PROJECT REQUIREMENTS.md` §3.3.1 pricing model UI, §2300 `pricing_type` / `min_price` schema intent
