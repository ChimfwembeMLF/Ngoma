# Implementation Plan: Client Design System Rollout

**Branch**: `003-client-design-system` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/003-client-design-system/spec.md`

## Summary

Replace the legacy dark indigo/cream styling on four client routes (Discover, Track, Auth, Dashboard) with the Airbnb-inspired light design system documented in `client/DESIGN.md`. Work is **client-only**: extend Tailwind with design tokens, add reusable UI primitives under `client/src/components/ui/`, wrap in-scope pages in a light layout shell, and refactor page markup to use shared components. No API, database, or routing changes.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (client workspace only)

**Primary Dependencies**:
- Client: React 18, Vite, Tailwind CSS 3, TanStack Query (existing hooks unchanged)
- Design reference: `client/DESIGN.md` (YAML tokens + component specs)

**Storage**: N/A — no schema changes

**Testing**: Manual visual validation per `quickstart.md` (VS-201–VS-205); optional Vitest snapshot tests for UI primitives if time permits (not required in spec)

**Target Platform**: Web SPA (mobile-first, 375px minimum)

**Project Type**: Yarn monorepo — changes confined to `client/`

**Performance Goals**: No regression in LCP on Discover; avoid layout shift on track cards (reserve aspect-ratio for cover art)

**Constraints**:
- Four pages only; out-of-scope routes keep current styling until a follow-up feature
- Preserve all existing behavior (search, playback, auth, navigation)
- Smallest correct diff — no shadcn/ui bootstrap unless component count justifies it

**Scale/Scope**: ~8 new UI components, 1 layout wrapper, Tailwind theme extension, 4 page refactors, 1 player restyle

**Reference**: `client/DESIGN.md`, `specs/003-client-design-system/contracts/`
