# Implementation Plan: shadcn/ui + Spotify Dark Redesign

**Branch**: `009-shadcn-spotify-redesign` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/009-shadcn-spotify-redesign/spec.md`

## Summary

Migrate the Ngoma client from **custom light-theme UI primitives** (003/004) to **shadcn/ui + Radix** with the **Spotify-inspired dark design system** in `client/DESIGN.md`. Work is **client-only**: bootstrap shadcn, remap Tailwind/CSS variables to Spotify tokens, introduce an app shell layout, replace all bespoke `Button`/`Input`/`Card` usage, and restyle every page and shared component. No API, database, or routing changes.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (client workspace only)

**Primary Dependencies**:
- Client: React 18, Vite 6, Tailwind CSS 3.4, TanStack Query (unchanged)
- **New**: shadcn/ui, Radix UI primitives, `class-variance-authority`, `tailwind-merge`, `lucide-react`, `tailwindcss-animate`
- Forms: React Hook Form + Zod + `@hookform/resolvers` (already installed)
- Design reference: `client/DESIGN.md` (Spotify dark theme)

**Storage**: N/A — no schema changes

**Testing**: Manual VS-901–VS-906 per `quickstart.md`; lint + build gates

**Target Platform**: Web SPA (mobile-first; DESIGN.md breakpoints 425–1280px+)

**Project Type**: Yarn monorepo — changes confined to `client/`

**Performance Goals**: No LCP regression on Discover; avoid layout shift on track cards (aspect-ratio preserved)

**Constraints**:
- Constitution requires shadcn/ui + Radix — closes gap vs current custom components
- Preserve all existing behavior (search, playback, auth, payments, playlists)
- Smallest correct diff per page — migrate imports, don't rewrite business logic
- Default dark theme only (`class="dark"` on root)

**Scale/Scope**:
- ~12 shadcn components installed
- Tailwind + `index.css` token overhaul
- 1 app shell layout replacing `DesignSystemLayout`
- ~15 pages + ~10 shared components refactored
- Remove legacy `components/ui/Button|Input|Card.tsx` after migration

**Reference**: `client/DESIGN.md`, constitution §V (Client Conventions), `PROJECT REQUIREMENTS.md` §1541
