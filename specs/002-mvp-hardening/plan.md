# Implementation Plan: MVP Hardening & Convergence

**Branch**: `002-mvp-hardening` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/002-mvp-hardening/spec.md`

## Summary

Close the gaps left after `001-platform-mvp` implementation: add an **Admin UI** for user management (FR-011), **extract audio duration** on upload, replace ILIKE discovery search with **PostgreSQL FTS**, and align **quickstart/dev config** with actual local ports (5433 Postgres, configurable API port). Extends existing `admin`, `tracks`, `media`, and `discovery` modules — no new top-level packages.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (api + client workspaces)

**Primary Dependencies**:
- API: NestJS 11+, TypeORM (existing); add `music-metadata` for duration parsing
- Client: React 18, Vite, TanStack Query (existing); new `AdminUsersPage`

**Storage**: PostgreSQL 15+ — one migration adding `search_vector tsvector` + GIN index on `tracks`; trigger or service-layer update on publish

**Testing**: Manual quickstart VS-1–VS-5 re-run; optional unit tests for duration parser and FTS query builder

**Target Platform**: Web (extends existing Ngoma stack)

**Project Type**: Yarn monorepo — `api/` + `client/`

**Performance Goals**: FTS search p95 < 200ms at 10k tracks; duration parse adds < 500ms to upload

**Constraints**: mako module layout; extend existing modules only; migrations required for FTS column

**Scale/Scope**: ~4 user stories, 1 migration, 2 API service updates, 2 client pages/components, docs updates

**Reference**: `specs/001-platform-mvp/`, `PROJECT REQUIREMENTS.md` §11.1 admin criteria
