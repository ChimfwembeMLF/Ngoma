# Implementation Plan: Ngoma Platform MVP

**Branch**: `001-platform-mvp` | **Date**: 2026-07-19 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/001-platform-mvp/spec.md`

## Summary

Build the Ngoma Phase 1 MVP as a **Yarn monorepo** (`api/` + `client/`) following the **mako** NestJS module layout. Deliver four user-facing slices: authentication, artist publishing, PawaPay mobile-money purchases, and listener discovery/playback. Backend modules map one domain per folder under `api/src/modules/`; the React SPA consumes REST APIs under `/api/v1/` via TanStack Query.

## Technical Context

**Language/Version**: TypeScript on Node.js 20+ (api + client workspaces)

**Primary Dependencies**:
- API: NestJS 11+, TypeORM, @nestjs/jwt, Passport, @nestjs/throttler, class-validator, BullMQ, @nestjs/swagger
- Client: React 18, Vite, Tailwind CSS, shadcn/ui, TanStack Query, React Hook Form, Zod, Howler.js

**Storage**: PostgreSQL 15+ (TypeORM migrations); Redis (cache, throttler, queues); Supabase Storage or S3 for audio/images

**Testing**: Jest (`api/test/`, `*.spec.ts`); Vitest + Testing Library (`client/src/**/*.test.tsx`)

**Target Platform**: Mobile-first web SPA + REST API (Zambia initial market)

**Project Type**: Yarn-workspace monorepo — `api/` (NestJS) + `client/` (React + Vite)

**Performance Goals**: API p95 < 500ms reads at MVP scale; track stream start < 3s on 3G

**Constraints**: mako module layout; `/api/v1/` prefix; TypeORM migrations only in shared envs; PawaPay sandbox first

**Scale/Scope**: ~10 NestJS modules, ~15 client pages/routes, 8 core DB entities for MVP

**Reference**: `PROJECT REQUIREMENTS.md`, `mako/api/`, `mako/client/`, `.specify/memory/constitution.md`
